# 任务模块功能

任务模块基于 Quartz 调度，用于管理签到、数据刷新、图标同步和其他自动化任务。它和通知、账号配置、任务日志联动，是系统自动运行的中枢。

![任务配置](assets/task-config.png)

## 任务配置

任务配置页展示任务名称、任务类型、通知渠道、通知配置、Cron 表达式、随机延时、创建人、下次执行时间、重试策略、状态和操作。

页面支持：

- 按任务名称、任务类型、状态筛选。
- 新增任务。
- 启用或暂停任务。
- 立即执行任务。
- 查看上次记录。
- 编辑任务。
- 删除任务。

## 当前任务类型

系统当前支持 9 类任务：

| 类型值 | 名称 |
| --- | --- |
| `demo` | 示例任务 |
| `huami_sport` | 华米运动 |
| `bilibili_sign` | BiliBili签到 |
| `atk_mall` | ATK商城 |
| `quark_cloud` | 夸克网盘 |
| `pt_sign` | PT通用 |
| `pt_subscription` | PT订阅 |
| `pt_site_data` | 站点数据 |
| `pt_favicon_sync` | 站点图标 |

账号配置页只展示需要账号凭据的任务类型；PT 订阅、站点数据、站点图标属于系统内部任务。

## 执行与重试

任务执行时会按配置的 Cron 表达式触发，也可以手动立即执行。随机延时用于错峰执行，减少多个任务同时访问外部服务。若任务开启重试，失败后会进入重试执行链路，并在任务日志中记录结果。

## 通知联动

任务可选择通知渠道和通知配置。当前通知渠道为 Bark。通知可按任务类型自动带上不同标题、图标、分组和跳转地址；任务也可以配置仅失败通知，减少无效消息。

## 日志查询

任务执行结果进入任务日志，可用于排查请求失败、Cookie 失效、站点解析变动、下载器推送异常等问题。排查任务问题时，建议先看最近一次执行记录，再根据异常信息定位账号配置、站点配置或外部服务状态。
